# Accessibility — Principles

## Overview

Non-negotiable requirements every design system must satisfy: WCAG 2.2 AA conformance, keyboard operability, focus visibility, touch target sizing, semantic structure, motion accommodation, and text readability.

---

## What's in this section

- **floor** — WCAG 2.2 AA as a constraint checklist. Color contrast (1.4.3, 1.4.11, dark mode, no raw values); keyboard navigation (2.1.1, 2.1.2, tab order); focus visibility (2.4.7, ring contrast, 2.4.11); touch targets (2.5.8 AA, 2.5.5 industry standard); semantic HTML and ARIA (4.1.2, accessible names, dynamic state); `prefers-reduced-motion`; text readability (16px, 90-char measure, 1.5 line height). Every requirement includes a verification method.
