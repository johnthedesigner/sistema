# Shopify Polaris — System Index

## Overview

Shopify Polaris is Shopify's design system for building admin interfaces and merchant-facing products. It covers visual design, component architecture, and — unusually among major design systems — an extensive and specific content design layer with detailed guidance on voice, tone, grammar, actionable language, and error messages.

The content guidelines within Polaris are particularly strong for UI copy in data-dense, task-oriented products. Polaris explicitly frames words as design material: "Very few interfaces make sense without content. But each word and every period adds noise to the experience." This philosophy — lean content, merchant-centered language, verb-first action — makes Polaris a primary reference for writing standards in B2B SaaS and commerce contexts.

When to reference this system: when writing UI copy for task-oriented or commerce products, when establishing verb-first action language standards, when defining error message patterns, or when grounding a voice guide in production practice.

## What's in this section

- **voice-and-tone** — Polaris's five content principles: lean content, human language, action-oriented writing, merchant-centered approach, and specific rules for UI copy.
- **grammar-and-mechanics** — Capitalization (sentence case), Oxford comma, period usage, contractions, number formatting, and American English conventions.
- **actionable-language** — Verb+noun patterns for buttons and actions, navigation label conventions, the minimal copy rule, and progressive disclosure.
- **error-messages** — Error message structure (heading, body, CTA), tone in error states, do/don't examples for common failure scenarios.
