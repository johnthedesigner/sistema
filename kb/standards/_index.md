## Overview

Normative reference material you cite rather than interpret. Standards documents define requirements and thresholds that design systems must meet — they are the source of truth for compliance decisions, not design inspiration.

**When to reach for this category:** accessibility audits, contrast checking, keyboard navigation requirements, ARIA pattern validation, and any work where the output needs to satisfy a published specification.

**Planned coverage:**

| Standard | Authority | Priority | Notes |
|---|---|---|---|
| WCAG 2.2 | W3C | High | Web Content Accessibility Guidelines — contrast, focus, interaction requirements |
| ARIA Authoring Practices Guide | W3C | High | Keyboard patterns, roles, and states for common UI components |
| APCA (Advanced Perceptual Contrast Algorithm) | Accessible Perceptual Contrast Algorithm | Medium | Next-gen contrast model being adopted in WCAG 3; significant departure from 4.5:1 ratio |
